\begin{abstract}
  Document layout analysis is a crucial step in the process of document understanding. As modern
  advancements in Optical Character Recognition \textit{(OCR)} have pushed the frontier of
  document understanding, there remain several unsolved problems that, once solved,
  \sout{will unlock new capacities for machine understanding of structured documents.}
  One such problem is verifying the source location of resultant text in parsed documents. OCR
  extracted text from a particular document has no structured paradigm for verifying the spatial
  location of origin. This is called \textit{alignment} between parsed text and its source location.
  We are proposing a framework that defines three data structures; \textit{Visual Tokens (VT), Text
  Tokens (TT), and a layout tree}, along with two algorithms- greedy and dynamic programming- to
  solve this alignment problem and achieve semantic provenance. In this paper we provide a formal
  definition of the alignment problem and propose the framework for semantic provenance. In future
  work we hope to tackle evaluation methods to measure *how* accurate the various algorithms are for
  achieving alignment. Our aim is to lay the groundwork for future research that furthers the development
  in document understanding.
\end{abstract}

\section{Overview of PaddlePaddle}
\label{sec:PaddlePaddle}

Our alignment framework builds on top of PaddleOCR~3.0~\cite{PaddleOCR30}, an open-source document understanding toolkit developed by Baidu on the PaddlePaddle deep learning platform. PaddleOCR~3.0 provides three solutions; we use two of them as preprocessing stages:

\begin{enumerate}
  \item \textbf{PP-OCRv5} --- a four-stage text detection and recognition pipeline that produces line-level and word-level visual tokens.
  \item \textbf{PP-StructureV3} --- a multi-model document parsing pipeline that recovers layout regions, content blocks, formulas, tables, and a markdown rendering of the page.
\end{enumerate}

Both pipelines share a common image preprocessing front-end (orientation classification and optional geometric unwarping) and together produce the inputs to our layout tree and alignment algorithms described in \Cref{sec:others,sec:alignment}.

\subsection{PP-OCRv5: Text Detection and Recognition}

The PP-OCRv5 pipeline~\cite{PaddleOCR30} processes a document page through four sequential stages: (1)~image preprocessing, which corrects page orientation and optionally unwarps photographed documents; (2)~text detection, which localizes text regions as bounding boxes; (3)~text-line orientation classification; and (4)~text recognition, which produces a text string and confidence score for each detected line. When word-level output is requested, the recognizer further segments each line into individual word boxes, each linked to its parent line. These outputs --- line and word bounding boxes with associated text --- constitute the \emph{visual tokens} consumed by our framework.

\subsection{PP-DocLayout: Layout Detection}

PP-DocLayout~\cite{PPDocLayout} is a layout detection model that localizes 23~categories of document elements, including paragraph titles, text blocks, images, tables, formulas, headers, footers, footnotes, seals, and charts. Each detected element is returned as a bounding box with a semantic label and a confidence score. These labels determine the \texttt{label} field that propagates through our layout tree (\Cref{sec:others}).

\subsection{PP-StructureV3: Document Parsing Pipeline}

PP-StructureV3~\cite{PaddleOCR30} composes PP-OCRv5 and PP-DocLayout into an end-to-end document parsing pipeline. Beyond detection, it applies specialized models for table recognition (outputting HTML), formula recognition (outputting \LaTeX), and chart parsing. A post-processing stage recovers reading order via an improved X-Y Cut algorithm, links captions to their figures or tables, and assembles the full page into structured JSON and Markdown.

Our framework consumes four outputs from this pipeline: the layout detection results for \emph{regions} and \emph{blocks} (with reading order), the formula recognition results, and the rendered \emph{Markdown}. The formal data model for these outputs is defined in \Cref{sec:formal_data_model}.

A \emph{document page} is a raster image $d \in \mathbb{R}^{H \times W \times 3}$. A \emph{bounding box} is a tuple $b = (x_0, y_0, x_1, y_1) \in \mathbb{Z}^4$ representing the axis-aligned rectangle $[x_0, x_1] \times [y_0, y_1]$, with area $|b| = (x_1 - x_0)(y_1 - y_0)$.

\subsubsection{OCR --- Visual Token Extraction}

The OCR step applies text detection and recognition to a page image, producing two granularities of visual tokens:
\begin{equation}
  \mathrm{OCR}(d) = (\mathcal{L}, \mathcal{W})
  \label{eq:ocr}
\end{equation}
where:
\begin{itemize}
  \item \textbf{Lines} $\mathcal{L} = \{(b_i, t_i, s_i)\}_{i=1}^{n_L}$, each consisting of a bounding box~$b_i$, a text string~$t_i$, and a confidence score $s_i \in [0, 1]$.
  \item \textbf{Words} $\mathcal{W} = \{(b_j, t_j, \ell_j)\}_{j=1}^{n_W}$, each consisting of a bounding box~$b_j$, a text string~$t_j$, and a parent line index $\ell_j \in \{1, \ldots, n_L\}$.
\end{itemize}
Words are pre-assigned to lines by the OCR engine via the foreign key~$\ell_j$.

\subsubsection{Layout Parsing --- Structural Extraction}

The layout parsing step applies PPStructureV3~\cite{PaddleOCRv3} to recover the semantic structure of the page:
\begin{equation}
  \mathrm{Layout}(d) = (\mathcal{R}, \mathcal{B}, \mathcal{F}, m)
  \label{eq:layout}
\end{equation}
where:
\begin{itemize}
  \item \textbf{Regions} $\mathcal{R} = \{(b_i, \lambda_i, s_i)\}_{i=1}^{n_R}$, coarse page zones with a semantic label~$\lambda_i$ and a detection score~$s_i$.
  \item \textbf{Blocks} $\mathcal{B} = \{(b_j, \lambda_j, o_j, c_j)\}_{j=1}^{n_B}$, content blocks with a label~$\lambda_j$, a reading order $o_j \in \mathbb{Z} \cup \{\bot\}$, and text content~$c_j$.
  \item \textbf{Formulas} $\mathcal{F} = \{(b_k, t_k, r_k)\}_{k=1}^{n_F}$, detected formulas with a \LaTeX\ string~$t_k$ and a region assignment~$r_k$.
  \item \textbf{Markdown} $m \in \Sigma^*$, a rendered markdown representation of the page content.
\end{itemize}

\subsubsection{Layout Tree}
\label{sec:layout_tree}

The layout tree $T = (V, E)$ is a four-level hierarchy that unifies the visual tokens from PP-OCRv5 with the structural outputs of PP-StructureV3 into a single rooted tree. Each node carries a bounding box, a semantic label, and textual content.

\paragraph{Intersection-over-child.}
To assign children to parents across levels, we define the \emph{intersection-over-child} (IoC) between a child box~$b_c$ and a candidate parent box~$b_p$:
\begin{equation}
  \mathrm{IoC}(b_c, b_p) = \frac{|b_c \cap b_p|}{|b_c|}
  \label{eq:ioc}
\end{equation}
where $b_c \cap b_p$ denotes the axis-aligned intersection rectangle (area~0 if the boxes are disjoint). Given a tolerance threshold $\tau \in [0, 1]$ (we use $\tau = 0.8$), each child is assigned to the parent that maximizes IoC, provided the overlap meets the threshold:
\begin{equation}
  \mathrm{parent}(c, P) = \underset{p \in P}{\arg\max}\;\mathrm{IoC}(b_c, b_p)
  \quad \text{s.t.} \quad \mathrm{IoC}(b_c, b_p) \geq \tau
  \label{eq:parent_assign}
\end{equation}
If no candidate meets the threshold, the child is unassigned ($\bot$). The threshold accommodates pixel-level noise in the bounding boxes predicted by the detection models.

\paragraph{Tree construction.}
The tree is built level by level, as summarized in \Cref{tab:tree_levels}. Levels~0--2 use the spatial IoC assignment of \Cref{eq:parent_assign}, while level~3 uses the foreign key~$\ell_j$ pre-computed by the OCR engine.

\section{Alignment Problem}
\label{sec:alignment}

Given a layout tree~$T$ and a reference text~$m$ (the Markdown produced by PP-StructureV3), the alignment problem asks: which substring of~$m$ corresponds to each node of~$T$? A solution assigns every node a contiguous span of text tokens such that (i)~sibling spans partition their parent's span without overlap, (ii)~the root interval covers all tokens, $\alpha(r) = [0, N)$, and (iii)~the total textual similarity between nodes and their assigned spans is maximized.

\subsection{Formal Definition}

Let $\mathbf{t} = (t_1, \ldots, t_N)$ be the tokenization of~$m$ using a TreebankWordTokenizer, with character spans $\phi(i) = (s_i, e_i)$ recording the position of each token in the original string. Define a similarity function
\begin{equation}
  \sigma : V \times \Sigma^* \to [0, 1]
  \label{eq:sim}
\end{equation}
that scores how well a node's content matches a candidate text span. We use the fuzzy token-sort ratio between the node's OCR content and the candidate string, normalized to $[0, 1]$.

An \emph{alignment} is a mapping
\begin{equation}
  \alpha : V \to \{(a, b) \mid 0 \leq a \leq b \leq N\}
  \label{eq:alpha}
\end{equation}
assigning each node~$v$ a token interval $[a, b)$ where $0 \leq a \leq b \leq N$, and by extension (via~$\phi$) a character span in~$m$. When $a = b$, the interval is empty --- the node is assigned no tokens. This arises naturally when a node's content has no textual counterpart in~$m$ (e.g.\ a figure or separator region).

\subsection{Constraints}

The alignment must respect the tree structure. For a node~$v$ with children $(c_1, \ldots, c_n)$ and assigned interval $\alpha(v) = [a, b)$:
\begin{enumerate}
  \item \textbf{Coverage.} The children's intervals partition the parent's interval: $a = k_0 \leq k_1 \leq \cdots \leq k_n = b$, with $\alpha(c_i) = [k_{i-1}, k_i)$.
  \item \textbf{Ordering.} The partition respects sibling order, preserving the reading order established during tree construction (\Cref{tab:tree_levels}).
  \item \textbf{Nesting.} Every descendant's interval is contained within its ancestor's interval.
\end{enumerate}

\subsection{Quality of Alignment}

For a node~$v$ with children $(c_1, \ldots, c_n)$ and interval $[a, b)$, the local alignment quality is the total similarity across the partition:
\begin{equation}
  Q(v) = \sum_{i=1}^{n} \sigma\left(c_i,\, \mathbf{t}_{k_{i-1}:k_i}\right)
  \label{eq:quality}
\end{equation}
The optimal partition maximizes $Q(v)$ over all valid split points $a = k_0 \leq \cdots \leq k_n = b$. The global alignment quality is the sum of $Q(v)$ over all internal nodes of~$T$.

\subsection{Algorithms}
\label{sec:alignment_algorithms}

We present three algorithms for solving the partition at each internal node, summarized in \Cref{tab:algo_comparison}. In all cases, let $n = |\text{children}|$ and $L = b - a$ be the length of the token interval.

\subsubsection{Dynamic Programming}

Define the recurrence:
\begin{equation}
  J(i, k) = \max_{0 \leq r \leq k} \Big[J(i-1, r) + \sigma\left(c_i,\, \mathbf{t}_{a+r:a+k}\right)\Big]
  \label{eq:dp}
\end{equation}
with base case $J(0, k) = \sigma(c_1,\, \mathbf{t}_{a:a+k})$ for $0 \leq k \leq L$. Traceback from $J(n-1, L)$ recovers the optimal split points. This guarantees a globally optimal partition for the node at cost $O(nL^2)$.

\subsubsection{Greedy}

The greedy algorithm processes children left to right. For each child~$c_i$ (except the last), it scans all candidate end positions $k \in [\mathrm{pos}, b]$ and selects the~$k$ that maximizes $\sigma(c_i, \mathbf{t}_{\mathrm{pos}:k})$. The last child receives the remaining interval. This runs in $O(nL)$ but may not find the global optimum.

\subsubsection{Hybrid}

The hybrid strategy selects the algorithm based on the interval length:
\begin{equation}
  \mathrm{Align}(v) =
  \begin{cases}
    \mathrm{DP}(v)     & \text{if } L \leq c \\
    \mathrm{Greedy}(v) & \text{if } L > c
  \end{cases}
  \label{eq:hybrid}
\end{equation}
where $c$ is a threshold (we use $c = 100$). Short intervals receive the exact DP solution; long intervals fall back to the greedy approximation to avoid the quadratic cost.

\centering
\caption{Comparison of alignment algorithms. $n$~denotes the number of children and $L$~the token interval length at each internal node.}
\label{tab:algo_comparison}
\begin{tabular}{lcl}
\toprule

\midrule

\paragraph{Recursive alignment.}
The full alignment is computed by recursing from the root with interval $[0, N)$. At each internal node, the chosen algorithm partitions the parent's interval among its children, then recurses into each child. Leaf nodes (words) inherit their interval directly. The result is the complete mapping $\alpha$ of~\Cref{eq:alpha}, from which character spans are recovered via~$\phi$.

\subsection{Reverse Alignment}

Given an alignment~$\alpha$ and a query character span $[q_s, q_e)$ in~$m$, the reverse alignment finds all layout tree nodes whose aligned character span overlaps the query, along with an overlap ratio indicating what fraction of each node's span is covered:
\begin{equation}
  \mathrm{ReverseAlign}(q_s, q_e) = \left\{\left(v,\, \frac{|[q_s, q_e) \cap [n_s, n_e)|}{n_e - n_s}\right) \;\middle|\; v \in V,\; \alpha(v) \mapsto [n_s, n_e),\; [q_s, q_e) \cap [n_s, n_e) \neq \emptyset\right\}
  \label{eq:reverse_align}
\end{equation}

This provides the inverse direction of provenance: given a passage of text, the reverse alignment identifies the visual tokens and layout regions on the page from which that passage originates, enabling spatial grounding of semantic queries; \textit{semantic provenance}.
