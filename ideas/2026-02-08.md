# An idea on alignment of 2D visual tokens with 1D markdown

The general problem is to align the visual _tokens_ with textual tokens.

## Definitions


**Page:**

> A page is a $2D$ image, given by $P\in\mathbf{R}^{H\times W\times 3}$.
>
> We will denote $H$ both as the height (integer) and the index range $[0, H-1]$ (similarly for $W$).
> A *page location* is given as the x-y coordinates corresponding to the row and column indices $(x, y)\in W\times H$.  A neighbourhood ${N}(x,y|m)$ is a square bounding box centered around $(x,y)$ defined by
> 
> $$
> N(x,y|m) = \{(i,j): x-{m}/{2}\leq i\leq x+{m}/{2}, y-m/2\leq j\leq y+m/2\}
> $$
>
>All arithmetics are done as integer arithmetics, and the bounds between $[0, H-1]$ and $[0, W-1]$ are omitted.
>
> The set of all page locations are defined as:
>
> $$\mathbf{Loc}(P) = W\times H$$

**Text:**

> The text representation is a list of word tokens given by $T\in\Sigma^*$ for some alphabet $\Sigma$.  A location in the text is an index $i\in[0, n-1]$ where $n=\mathrm{length}(T)$.  A neighbourhood $N(i|m)$ is given as the range $[i-m/2, i+m/2]$.
>
> The set of all *text locations* are given by:
> $$\mathbf{Loc}(T) = [0, n-1]$$

**Visual Tokens**

Visual tokens are rectangles with textual annotation.  Specifically,
$$
V = \{(r_i, t_i): r\in\mathbb{N}^4, t\in\Sigma^*\}
$$

## Alignment

$\newcommand{\Loc}{\mathbf{Loc}}$

We want to align $\Loc(P)$ with $\Loc(T)$ using the visual tokens.  More concretely, we want to define a similarity function:

$$ \mathrm{sim} : \Loc(P)\times\Loc(T)\to \mathbf{R} $$

Given a page location $(x,y)$ and a text location $i$, we have the neighbourhoods of $N_1 = N((x,y)|k_1)$ and $N_2=N(i|k_2)$ where $k_1$ is the neighbourhood size in the page, and $k_2$ is the neighbourhood size in the text.  We will denote:

- $t_1 = \mathrm{text}(N_1) = \{t_i: (r_i, t_i)\in V, r_i\cap N_1\not=\emptyset\}$
- $t_2 = \mathrm{text}(N_2) = T[N_2]$, i.e. words in $T$ that lie inside the neighbourhood of $N_2$.

We need to convert $t_1$ and $t_2$ into two bags of 3-grams at the character level.  Denote 
$$
\mathrm{ngram}_3:\Sigma^* \to \mathbf{BAG}[\Sigma^3]
$$
as the ngram tokenizer which first pads the input with 2 letters `$$`, and then performs 3-gram tokenization.

Finally the similarity is defined as:

$$
\mathrm{sim}((x, y), i | (k_1, k_2)) = \mathrm{Jaccard}(\mathrm{ngram}_3(t_1), \mathrm{ngram}_3(t_2))
$$
