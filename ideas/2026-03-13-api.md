# RESTful API

<!-- Implementation notes: use FastAPI. The data root is a directory passed at server startup
     (previously `--data` flag via `ppx server start`). Each document is a subdirectory of the
     data root; each page is a numbered subdirectory inside the document directory. -->

## GET `/health` - check if the server is running

The response should be:

```json
{
    "status": "ok"
}
```

<!-- Trivial. Return a hardcoded JSON response. -->

## GET `/v1/ppx/documents` - list all documents

The response should be:

```json
{
    "documents": [
        {
            "filename": "document1",
            "pages": 2
        },
        {
            "filename": "document2",
            "pages": 3
        }
    ]
}
```

<!-- Enumerate subdirectories of the data root. `pages` count = number of numeric
     subdirectories that contain at least `regions.parquet` (the `_is_page_dir` check
     already exists in cli/commands/ocr.py). -->

## GET `/v1/ppx/documents/{filename}/pages` - get document pages

The response should be:

```json
{
    "pages": [
        {
            "page_index": 0,
            "width": 100,
            "height": 100
        },
        {
            "page_index": 1,
            "width": 100,
            "height": 100
        }
    ]
}
```

<!-- Width/height can be read from `np_page.png` via PIL without loading the full pixel
     data (PIL reads image headers lazily with `Image.open`). -->

## GET `/v1/ppx/documents/{filename}/pages/{page_index}/image`

get document page image

The response should be of content type "image/png".

## GET `/v1/ppx/documents/{filename}/pages/{page_index}/visual_tokens`

get document page visual tokens.  The response should be of content type "application/json".  The response is the serialization of the union of the `regions`, `blocks`, `lines`, and `words` dataframes.

<!-- Load and concat: `line_tokens.parquet`, `word_tokens.parquet`, `regions.parquet`,
