I want to implement a storage system for the instances of

- @VisualTokenLayer
- @LayoutLayers

We store shared field `np_page` only once as png file, and all other fields are dataframes which are stored as parquet files.

The storage API is given as:

```python
@dataclass
class WriteStatus:
    path: Path # path to the file
    size: int # size in bytes of the file
    message: str # success or error message

# returns a generator of WriteStatus for each file written.
# this is yield back to the caller for synchronous progress reporting.
def store(
    path: Path,
    visual_token_layer: VisualTokenLayer,
    layout_layers: LayoutLayers,
    overwrite: bool = False,
) -> Iterable[WriteStatus]:
    ...

# load the layers.  The path is assumed to be of the form ".../<page_index>"
# page_index is just a string representation of an integer.
def load(path: Path) -> tuple[VisualTokenLayer, LayoutLayers]:
    ...
```

The storage layout is given as:

```
<path>/<page_index>
 - np_page.png
 - visual_tokens.parquet
 - layout.parquet
 - blocks.parquet
 - formulas.parquet
```

If `overwrite` is `True`, the existing files are overwritten. If `overwrite` is `False`, the existing files are not overwritten.

No need to dataframe schema validation.
